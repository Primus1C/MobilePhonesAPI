# Ресурс `Состояние` регистра `СостоянияИнвойсовПродажи`

Сводка по конфигурации **ИТДБ_Торговля_8_3**: когда и где устанавливаются значения ресурса **`Состояние`** регистра сведений **`СостоянияИнвойсовПродажи`**.

Запись выполняется **только** через `РаботаСИнвойсами.ЗаписатьСтатусИнвойсаПродажи` (плюс ручное редактирование записи регистра в интерфейсе). Прямых записей из **`_ВозвратОтПокупателя`** нет.

Допустимые значения перечисления **`СтатусыИнвойсаПродажи`**: `Зарегистрирован`, `Изменен`, `Собирается` (синоним «Собирается (заблокирован)»), `ГотовКОтгрузке`, `Отгружен`, `Отменен`.

---

## Общая логика записи

В `РаботаСИнвойсами.ЗаписатьСтатусИнвойсаПродажи`:

- Если **`Статус` не заполнен** (пустая ссылка / параметр не передан), **`Состояние` не меняется** — обновляются только переданные API-ресурсы (`Заблокирован`, `ОтмененЗакачиком`, `ГотовКОтгрузке`).
- Если новое значение **совпадает** с текущим и нет изменений API-флагов — **запись не выполняется**.
- При изменении `Состояние` заполняются **`Инициатор`**, **`Зарегистрировано`**, при необходимости **`КемИзменено`**.

---

## 1. Документ `ИнвойсПродажи`

Модуль объекта: `Documents/ИнвойсПродажи/Ext/ObjectModule.bsl` (блок в `ОбработкаПроведения` и `ОбработкаУдаленияПроведения`).

| Значение `Состояние` | Когда | Условия |
|----------------------|--------|---------|
| **Зарегистрирован** | Проведение | `РежимЗаписи = Проведение`, документ **не** помечен как изменённый (`ИзмененИнтерактивно` / `ИзмененВызовомAPI`), и (**нет** записи в РС **или** текущее `Состояние = Отменен`). |
| **Изменен** | Проведение | `РежимЗаписи = Проведение` и `ДокументИзмененПередПроведением()` = Истина: форма была модифицирована **или** в доп. свойствах **`ИзмененВызовомAPI`** (PATCH/PUT/DELETE через StockAPI). |
| **Отменен** | Снятие проведения | `ОбработкаУдаленияПроведения` — **всегда** при отмене проведения инвойса. |
| **Отменен** | Проведение (ветка) | В `ОбработкаПроведения` при `РежимЗаписи = ОтменаПроведения` — дублирует идею снятия проведения (на практике снятие идёт через `ОбработкаУдаленияПроведения`). |
| **Отменен** | Проведение (ветка) | В `ОбработкаПроведения` при `РежимЗаписи = Запись` и **пометка удаления**, если текущее состояние ≠ `Отменен`. |
| **Зарегистрирован** | Проведение (ветка) | Там же: `Запись`, **снята** пометка удаления, текущее `Состояние = Отменен`. |

Инициатор записи: сам **`ИнвойсПродажи`**. `КемИзменено` — из `ДополнительныеСвойства.КемИзменено` (API/форма) или сотрудник текущего пользователя.

**Важно для API:** PATCH/PUT/DELETE сначала **снимают проведение** → срабатывает **`ОбработкаУдаленияПроведения`** → **`Отменен`**, затем при повторном проведении с флагом API → **`Изменен`**.

**DELETE /orders (все позиции):** только **пометка на удаление** в `StockAPI_Заказы.ПометитьИнвойсНаУдаление` — **отдельной записи в РС нет**; `Отменен` по пометке сработает только если пройдёт ветка «Запись + пометка» в модуле объекта (для проведённого документа без снятия проведения статус может **не** обновиться).

## 2. Документ `РасходнаяНакладная`

Модуль объекта: `Documents/РасходнаяНакладная/Ext/ObjectModule.bsl` — процедуры `ОбработкаПроведения`, `ОбработкаУдаленияПроведения`, `УстановитьСтатус`.

Запись только если **`ДокументОснование`** типа **`ИнвойсПродажи`**. Инициатор — **ссылка на расходную накладную**.

| Значение `Состояние` | Когда | Условия |
|----------------------|--------|---------|
| **Отгружен** | Проведение накладной | Реквизит **`Окончательно = Истина`**. |
| **Собирается** | Проведение накладной | **`Окончательно = Ложь`**. |
| **ГотовКОтгрузке** | Отмена проведения накладной | `ОбработкаУдаленияПроведения`, если проверки не установили `Отказ` (для неадминистратора — нельзя отменить «окончательную» или непустую накладную). Для **администратора** — та же установка статуса без отмены проведения (ранний выход). |

**StockAPI `POST /delivery`:** при заполнении накладной выставляется **`Окончательно = Истина`** → при проведении у заказа **`Отгружен`**, не `Собирается`.

## 3. StockAPI (HTTP)

| Метод | Меняет `Состояние`? | Что происходит |
|-------|---------------------|----------------|
| **POST /orders** | Да | Проведение нового `ИнвойсПродажи` → обычно **`Зарегистрирован`**. Затем опционально `readyToShip` — **без** смены `Состояние`. |
| **PATCH / PUT / DELETE /orders** | Да | Снятие проведения → **`Отменен`**; проведение с **`ИзмененВызовомAPI`** → **`Изменен`**. `readyToShip` только в PATCH — отдельно, без смены `Состояние`. |
| **POST /delivery** | Да (косвенно) | Проведение РН с **`Окончательно`** → **`Отгружен`**. |
| **PATCH /delivery** | **Задумано `Собирается`, по коду — нет** | Вызов `ЗаписатьСтатусИнвойсаПродажи` **без** заполненного `Статус`; в 7-й параметр передана **`ТекущаяДатаСеанса()`** вместо булева `ЗаблокированАПИ`. **`Состояние` не устанавливается в `Собирается`**; возможна некорректная запись ресурса `Заблокирован`. Ответ API берёт `status` из регистра **после** вызова. |
| **POST /status** | **Нет** | `Статус = ПустаяСсылка()` — меняются только **`Заблокирован` / `ОтмененЗакачиком` / `ГотовКОтгрузке`**, не `Состояние`. |
| **GET***, **POST /return** | Нет | Только чтение / документ возврата. |

---

## 4. `_ВозвратОтПокупателя`

Обращений к `ЗаписатьСтатусИнвойсаПродажи` / `СостоянияИнвойсовПродажи` **нет**. Возврат **не меняет** `Состояние` заказа в этом регистре (в API в `GET /orders` могут подтягиваться `returnNumber` / `returnDate` из документа возврата).

## Сводная таблица «кто выставляет какое `Состояние`»

| `Состояние` | Основные источники |
|-------------|-------------------|
| **Зарегистрирован** | Первое проведение `ИнвойсПродажи` (в т.ч. **POST /orders**); восстановление после снятия пометки удаления (ветка в модуле объекта). |
| **Изменен** | Повторное проведение после правок (форма или **PATCH/PUT/DELETE API**). |
| **Собирается** | Проведение **неокончательной** `РасходнаяНакладная` по заказу; **не** через **POST /delivery** API. |
| **ГотовКОтгрузке** | Отмена проведения РН; команда «Установить статус»; форма РС. |
| **Отгружен** | Проведение **окончательной** РН (**POST /delivery** всегда окончательная). |
| **Отменен** | Снятие проведения `ИнвойсПродажи` (в т.ч. перед правкой через API); пометка удаления (если сработает ветка в модуле); команда/форма. |

## Что API показывает как `status`, но это не `Состояние`

В JSON **`blocked`**, **`canceled`**, **`readyToShip`** — это **другие ресурсы** того же регистра (`Заблокирован`, `ОтмененЗакачиком`, `ГотовКОтгрузке`), они **не равны** значению перечисления в `Состояние`.

Например, **`POST /status` с `blocked=true`** не переводит `Состояние` в `Собирается`; для этого в UI/логике задумывались проведение неокончательной РН или **PATCH /delivery** (с оговоркой по некорректному вызову в `StockAPI_Заказы.ЗаблокироватьРасходнуюНакладную`).
