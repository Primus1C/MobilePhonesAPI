Инструкция: тестовый сервер вебхуков 1С
Базовый URL
http://37.44.215.33:3099
Главный endpoint: POST /webhooks/1c
т.е. все запросы кидаем на http://37.44.215.33:3099/webhooks/1c
Авторизация (Bearer) :
токен - jkdsfiu840320-eFEW324klsdfoi234
Без ключа (или с неверным) сервер вернёт 401
`GET /health`, `GET /schema`, `GET /history` — без авторизации
Типы событий (`eventType`)
`Остаток`
`Цена`
`Наличие` | Появились новые позиции → запросить карточки; `available: false` → снять с наличия | «Ок, запрашиваю новые позиции…» | , этот кейс если вдруг надо убрать из продажи какие либо позиции , но они лежат на складе.
Обязательно в теле:
- `eventType` — одно из значений выше
- `items` — непустой массив; у каждой позиции нужен `itemCode`
Остальные поля (`batchId`, `createdAt`, любые доп. поля из 1С) допускаются.

Другие эндпоинты
| `GET` | `/health` | Жив ли сервер |
| `GET` | `/schema` | Контракт и примеры полей |
| `GET` | `/history?limit=50` | Последние принятые вебхуки |
| `GET` | `/history/batch/:batchId` | Найти пакет по `batchId` |
Ошибки:
401 - Нет `Authorization`, неверный ключ, или не формат `Bearer …`
400 - Невалидный JSON
400 -  Некорректное тело вебхука (валидация)
404 -  Несуществующий путь или `batchId` не найден в `/history/batch/:id`
500 - Внутренняя ошибка сервера
